Whitepaper

Egalitarianism (EGA) · v0.1 draft · living document aligned with shipped Core parameters. Not financial advice.

1. Vision

Money and mining power should not require elite hardware or insider allocation.

Egalitarianism aims to:

  • Keep permissionless access to create and transfer value
  • Keep mining open to CPUs, consumer GPUs, phones, and single-board computers
  • Avoid premine / founder stash at genesis
  • Resist capture by a single hardware class

Short name / ticker: EGA.

2. Design principles

  1. Fair launch — no premine; genesis coinbase value is zero.
  2. Multi-door mining — three algorithms, three hardware paths.
  3. Simple emission — Bitcoin-style halvings, transparent math.
  4. Independent difficulty — MultiShield-style per-algo adjustment.
  5. Honesty — capital still matters; the design raises the bar for monopoly, it does not erase it.

5. Node software

Egalitarianism Core (egad, ega-cli, ega-qt) is a full node derived from DigiByte/Bitcoin Core architecture, customized for these rules. Solo mining via RPC; GBT for external miners.

Source: github.com/Egalitarianism-EGA/ega

6. Roadmap

Phase A — Live chain

  • Consensus + triple PoW + genesis
  • Solo mining (CPU + GPU Verthash path)
  • 24/7 seed node(s) and public addnode list
  • Linux binary release

Phase B — Community

  • Independent nodes
  • Public explorer and multi-algo pool
  • Clear user mining docs

Phase C — Maturity

  • Signed releases
  • Optional RandomX full-mem mode
  • Algo upgrade policy if ASICs appear

7. Risks

  • Low hashrate early → 51% is easier; multi-algo helps, does not eliminate risk
  • Software bugs in a young fork
  • Pool centralization if one pool dominates an algo
  • Verthash miner mismatch if the wrong dataset is used
